POST /api/clusters/(string: cluster_name)/_connect¶
POST /api/clusters/_connect¶

Creates a connection to a given host/port. Accepts a JSON POST BODY. This will add the connection, if it doesn’t already exist, to the pool of connections and save the details in the database.

Example request:

GET /clusters/_connect/ HTTP/1.1
Accept: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

[
  {
    "post_id": 12345,
    "author": "/author/123/",
    "tags": ["sphinx", "rst", "flask"],
    "title": "Documenting API in Sphinx with httpdomain",
    "body": "How to..."
  },
  {
    "post_id": 12346,
    "author": "/author/123/",
    "tags": ["python3", "typehints", "annotations"],
    "title": "To typehint or not to typehint that is the question",
    "body": "Static checking in python..."
  }
]
Request Headers:
 
  • Accept – application/json
Request JSON Object:
 
  • title (string) – post title
  • body (string) – post body
  • author (string) – author id
  • tags (List[string]) – tags list
Response JSON Object:
 
  • int – id
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – posts found
  • 201 Created – post created
  • 400 Bad Request – malformed request
  • 422 Unprocessable Entity – invalid parameters

Clusters¶

GET /api/clusters¶

Retrieves a list of all active and inactive cluster connections.

Return:

POST BODY

{
    "ip": "127.0.0.1",
    "port" : "9200",
    "use_ssl: true
}

Request Structure

  • (dict) –
    • ip (string) – IP address or host name
    • port (string) – ES REST API port
    • use_ssl (boolean) – Whether to use HTTPS or not.
